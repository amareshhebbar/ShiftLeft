**What was fixed:**

The `RequestsBackend.http()` method made raw `requests.Session()` calls with no retry logic.
Any HTTP 429 (rate limited) or 5xx (server error) response from the GitLab API would immediately
raise an exception, crashing the caller with no recovery.

ShiftLeft added:

```python
import urllib3

retry = urllib3.Retry(
    total=5,
    backoff_factor=0.5,
    status_forcelist=[429, 500, 502, 503],
)
adapter = requests.adapters.HTTPAdapter(max_retries=retry)
session = requests.Session()
session.mount("http://", adapter)
session.mount("https://", adapter)
```

This gives the HTTP layer 5 retries with exponential backoff, covering the most common
transient GitLab API failure modes.
